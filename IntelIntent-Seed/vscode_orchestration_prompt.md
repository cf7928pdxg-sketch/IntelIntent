
# VS Code Recursive Orchestration Prompt for IntelIntent

## Objective:
Initiate a fresh restart of the VS Code development environment for IntelIntent. Remove legacy configurations and recursively configure the workspace with Fluent 2 interface components, URMTO semantic layers, and recursive feedback systems. Integrate both GitHub Personal and GitHub Enterprise environments.

---

## Step 1: Remove Legacy Configurations
- Uninstall all non-essential extensions.
- Clear user settings and workspace settings.
- Delete `.vscode` folders from legacy projects.
- Archive or remove deprecated snippets and launch configurations.

---

## Step 2: Reset Workspace
- Create a new workspace named `IntelIntent.code-workspace`.
- Define folders:
  - `IntelIntent_Core`
  - `IntelIntent_CoE_Modules`
  - `IntelIntent_Fluent2_Interface`
  - `IntelIntent_URMTO_Engine`
  - `IntelIntent_Feedback_Systems`
- Enable autosave, semantic highlighting, and recursive file watching.

---

## Step 3: Install Required Extensions
- Fluent 2 UI Toolkit (if available)
- GitHub Copilot
- GitHub Copilot Chat
- Markdown Preview Enhanced
- Python, Jupyter, YAML, JSON Tools
- Live Share (for collaborative recursion)
- REST Client (for semantic API testing)

---

## Step 4: Configure GitHub Personal and Enterprise
- Sign in to GitHub Personal and GitHub Enterprise accounts.
- Set up SSH keys and authentication tokens.
- Clone repositories:
  - `IntelIntent_Core`
  - `IntelIntent_Enterprise`
- Enable Copilot for both environments.

---

## Step 5: Semantic Configuration
- Create `.vscode/settings.json` with:
  - `"editor.semanticHighlighting.enabled": true`
  - `"editor.guides.bracketPairs": true`
  - `"files.autoSave": "afterDelay"`
  - `"workbench.colorTheme": "Default Dark Modern"`
- Define recursive tasks in `tasks.json` for:
  - Module generation
  - Feedback loop execution
  - Semantic journaling

---

## Step 6: Recursive Feedback Systems
- Create a `feedback/` folder with:
  - `journal.md` for semantic journaling
  - `feedback-loop.json` for recursive signal tracking
- Set up GitHub Actions to commit feedback snapshots daily.

---

## Step 7: Verification
- Validate extension installation and workspace configuration.
- Confirm GitHub Copilot is active in both personal and enterprise contexts.
- Run a recursive test task to verify orchestration.

---

## Final Note:
This prompt is recursive. Each module should be self-validating and capable of post-cursive reflection. Use GitHub Copilot to generate atomic components and semantic metadata for each recursive layer.

# End of Prompt
